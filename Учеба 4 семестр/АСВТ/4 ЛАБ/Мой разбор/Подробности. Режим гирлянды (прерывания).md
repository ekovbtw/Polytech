#### Зачем нужен garland_tick и GARLAND_DIV

---

#### Проблема

Прерывание от таймера срабатывает 800 раз в секунду. Если двигать светодиоды каждый раз — они будут мигать так быстро, что глаз вообще ничего не увидит. Нужно замедлить.

---

#### Решение — считаем прерывания

Заводим счётчик garland_tick. Каждый раз когда срабатывает прерывание — прибавляем 1. Когда счётчик дошёл до 64 — только тогда двигаем светодиоды. Потом сбрасываем счётчик и начинаем снова.

Визуально:

```
Прерывание:  1   2   3  ...  63   64  → двигаем светодиоды, сбрасываем в 0
             ↑   ↑   ↑        ↑    ↑
         +1  +1  +1          +1  сброс

Прерывание:  1   2   3  ...  63   64  → двигаем снова
```

---

#### Что делает этот код (замедляет мерцание светодиодов)

```
garland_tick++;
if (garland_tick >= GARLAND_DIV) {
    garland_tick = 0;
    // здесь двигаем светодиоды
}
```

Шаг 1 — каждое прерывание прибавляем 1 к счётчику

Шаг 2 — проверяем: счётчик дошёл до 64?

Шаг 3 — если да: сбрасываем счётчик в 0 и делаем шаг гирлянды

Шаг 4 — если нет: просто выходим из прерывания, ничего не делаем

---

#### Почему именно 64

GARLAND_DIV = 64 — это просто выбор разработчика. При 800 прерываниях в секунду и делителе 64 получается:

```
800 / 64 = 12.5 шагов гирлянды в секунду
```

Это удобное число — потому что дальше в коде нужно было получить ровно 25 шагов за 2 секунды, а 12.5 × 2 = 25.

---

По сути garland_tick — это просто таймер внутри таймера. Первый таймер тикает быстро (800 раз/сек), а garland_tick замедляет его в 64 раза для гирлянды.
