#### Что такое потенциометр и зачем он здесь

Потенциометр — это крутилка. Когда ты её поворачиваешь, она меняет напряжение на своём выходе от 0В до 5В. Но микроконтроллер понимает только цифры, не напряжение. Вот тут и нужен АЦП — он превращает аналоговое напряжение в число.

---

#### Что делает АЦП

АЦП — Аналогово-Цифровой Преобразователь. Он читает напряжение на ножке PA5 и выдаёт число от 0 до 1023. Это 10 бит.

```
Напряжение 0В   →  число 0
Напряжение 2.5В →  число 512
Напряжение 5В   →  число 1023
```

Формула из методички:

```
ADC = Vin × 1024 / Vref
```

Где Vref = 5В (питание). То есть АЦП делит весь диапазон напряжений на 1024 равные части.

---

#### Как настроен АЦП в коде

c

```c
void ADC_init(void) {
    ADMUX  = (1<<REFS0) | (1<<MUX2) | (1<<MUX0);
    ADCSRA = (1<<ADEN) | (1<<ADSC) | (1<<ADATE) | (1<<ADPS2) | (1<<ADPS1) | (1<<ADIE);
}
```

Разбираем ADMUX побитово:

- REFS0 = 1, REFS1 = 0 — опорное напряжение берём от AVCC (5В)
- MUX2 = 1, MUX0 = 1, остальные MUX = 0 — выбираем канал ADC5, это ножка PA5

Разбираем ADCSRA побитово:

- ADEN = 1 — включаем АЦП
- ADSC = 1 — запускаем первое преобразование
- ADATE = 1 — режим авто-запуска, АЦП работает непрерывно само
- ADPS2 = 1, ADPS1 = 1 — делитель частоты 64, чтобы АЦП тактировался на 8000000/64 = 125 кГц (методичка говорит нужно 50-200 кГц для точности)
- ADIE = 1 — разрешить прерывание когда АЦП закончил замер

---

#### Как приходит результат — прерывание АЦП

Когда АЦП заканчивает замер — автоматически вызывается прерывание:

c

```c
ISR(ADC_vect) {
    adc_value = ADC;
    adc_ready = 1;
}
```

ADC — это специальный регистр где лежит результат. Просто читаем его и сохраняем. Флаг adc_ready = 1 говорит главному циклу что данные готовы.

---

#### Как из adc_value получается pb

Это происходит в главном цикле while(1), когда mode == 1:

c

```c
adc_sum += adc_value;
adc_count++;

if (adc_count >= 16) {
    unsigned int adc_avg = adc_sum / 16;
    adc_sum = 0;
    adc_count = 0;

    uint8_t brightness = (uint8_t)(adc_avg >> 2);
    OCR2  = brightness;
    OCR1B = brightness;

    pb = adc_avg * 8 / 1024 + 1;
    if (pb > 8) pb = 8;
}
```

Сначала накапливаем 16 замеров и усредняем — чтобы убрать дрожание значения при неподвижной крутилке.

Потом из среднего значения считаем pb:

```
adc_avg = 0    → pb = 0*8/1024 + 1 = 1  (минимум)
adc_avg = 512  → pb = 512*8/1024 + 1 = 5
adc_avg = 1023 → pb = 1023*8/1024 + 1 ≈ 8 (максимум)
```

Крайнее левое положение крутилки — pb = 1 (медленно). Крайнее правое — pb = 8 (быстро).

---

#### Зачем brightness и >> 2

АЦП даёт 10-битное число (0-1023). ШИМ принимает 8-битное (0-255). Нужно преобразовать.

Сдвиг вправо на 2 = деление на 4:

```
1023 >> 2 = 255   ✓
512  >> 2 = 128   ✓
0    >> 2 = 0     ✓
```
