#### Сначала — обычный сдвиг, чтобы понять проблему

В C есть оператор сдвига влево — <<. Например:

```
0b00001111 << 1 = 0b00011110  ✓
0b00001111 << 2 = 0b00111100  ✓
0b10000000 << 1 = 0b00000000  ✗ — бит потерялся!
```

При обычном сдвиге биты, которые выходят за пределы байта, просто теряются. Для гирлянды это плохо — огонь уйдёт за край и исчезнет.

---

#### Что такое циклический сдвиг

Циклический сдвиг — это когда бит, который вышел за левый край, появляется с правой стороны. Как Пакман — вышел слева, появился справа.

Пример с hb = 1:

```
Шаг 0:  0b10000001
Шаг 1:  0b00000011  ← бит из позиции 7 перешёл в позицию 0
Шаг 2:  0b00000110
Шаг 3:  0b00001100
...и так по кругу
```

---

#### Как это реализовано в коде

c

```c
void crcl(uint8_t *cur, int value) {
    *cur = ((*cur << value) | (*cur >> (8 - value))) & 0xFF;
}
```

Разберём по частям на примере: cur = 0b10110000, value = 2

Часть 1 — сдвигаем влево на 2:

```
0b10110000 << 2 = 0b11000000  (биты 7 и 6 потерялись)
```

Часть 2 — сдвигаем вправо на (8 - 2) = 6:

```
0b10110000 >> 6 = 0b00000010  (биты 7 и 6 теперь справа)
```

Часть 3 — склеиваем через OR:

```
  0b11000000
| 0b00000010
= 0b11000010
```

Часть 4 — маска & 0xFF, чтобы обрезать лишние биты (потому что в C при сдвиге int может расширяться):

```
0b11000010 & 0xFF = 0b11000010
```

---

#### Визуально как работает оборачивание

```
До сдвига:    [ 1 0 1 1 0 0 0 0 ]
                7 6 5 4 3 2 1 0

После << 2:   [ 1 1 0 0 0 0 0 0 ]  ← левая часть
                7 6 5 4 3 2 1 0

После >> 6:   [ 0 0 0 0 0 0 1 0 ]  ← "потерянные" биты вернулись справа
                7 6 5 4 3 2 1 0

OR итог:      [ 1 1 0 0 0 0 1 0 ]
                7 6 5 4 3 2 1 0
```

#### Где вызывается crcl в коде

В прерывании таймера, для каждого порта отдельно:

c

```c
if (++count_b >= (25 / pb)) {
    count_b = 0;
    crcl(&cur_b, hb);  // сдвигаем маску порта B на hb бит
}
```

После сдвига вызывается update_garland() — и новое значение cur_b выводится на PORTB.
