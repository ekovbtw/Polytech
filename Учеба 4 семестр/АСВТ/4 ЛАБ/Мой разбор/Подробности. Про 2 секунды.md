#### Считаем частоту тиков таймера

Таймер T0 настроен так:

- Частота процессора — 8 000 000 Гц
- Делитель — 64
- OCR0 = 154, значит таймер считает до 155

Частота прерываний таймера:

```
8 000 000 / (64 × 155) ≈ 800 раз в секунду
```

То есть прерывание от таймера срабатывает примерно 800 раз в секунду.

---

#### Считаем тики гирлянды

В коде есть GARLAND_DIV = 64. Это значит, что один шаг гирлянды происходит не каждое прерывание, а каждые 64 прерывания:

```
800 / 64 = 12.5 тиков гирлянды в секунду
```

---

#### Откуда взялось число 25 в коде

В прерывании таймера для порта B написано:

```
if (++count_b >= (25 / pb))
```

Смысл такой — накапливаем тики гирлянды до порога 25/pb, потом делаем сдвиг.

Если pb = 1, порог = 25. Значит ждём 25 тиков гирлянды:

```
25 тиков / 12.5 тиков в секунду = 2 секунды
```

Если pb = 2, порог = 12. Ждём 12 тиков:

```
12 / 12.5 ≈ 1 секунда → то есть 2 сдвига за 2 секунды
```

Если pb = 8, порог = 3. Ждём 3 тика:

```
3 / 12.5 ≈ 0.24 секунды → то есть 8 сдвигов за 2 секунды
```

---

#### Итог

Число 25 — это просто количество тиков гирлянды за 2 секунды (12.5 × 2 = 25). Разработчик специально подобрал параметры таймера так, чтобы получилось ровно 25 тиков за 2 секунды. Тогда pb удобно читается как "количество сдвигов за 2 секунды".
