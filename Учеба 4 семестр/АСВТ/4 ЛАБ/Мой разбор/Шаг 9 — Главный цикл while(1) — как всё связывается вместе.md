#### Зачем нужен while(1) если есть прерывания

Прерывания — это реакция на события (таймер тикнул, кнопку нажали, АЦП закончил замер). Но между событиями процессор должен что-то делать. Вот для этого и нужен главный цикл — он крутится бесконечно и обрабатывает данные когда они готовы.

В этой программе главный цикл делает одно — обрабатывает значение потенциометра в режиме настройки.

---

#### Полный главный цикл с разбором

c

```c
while(1) {
    if (mode == 1) {
```

Сначала проверяем — мы вообще в режиме настройки? Если mode == 0, весь цикл просто пропускается и сразу повторяется снова.

c

```c
        if (adc_ready) {
            adc_ready = 0;
```

Флаг adc_ready выставляет прерывание АЦП когда новый замер готов. Сразу сбрасываем его в 0, чтобы не обработать одно и то же значение дважды.

c

```c
            static unsigned int adc_sum = 0;
            static uint8_t adc_count = 0;

            adc_sum += adc_value;
            adc_count++;
```

Ключевое слово static — эти переменные не сбрасываются при каждом входе в цикл, они живут всё время работы программы. Накапливаем сумму замеров и считаем сколько накопили.

c

```c
            if (adc_count >= 16) {
                unsigned int adc_avg = adc_sum / 16;
                adc_sum = 0;
                adc_count = 0;
```

Когда накопили 16 замеров — усредняем. Это убирает дрожание значения когда крутилка стоит на месте. После усреднения сбрасываем накопители.

c

```c
                uint8_t brightness = (uint8_t)(adc_avg >> 2);
                OCR2  = brightness;
                OCR1B = brightness;
```

Преобразуем 10-битное значение АЦП в 8-битную яркость (делим на 4 сдвигом). Записываем в оба ШИМ регистра — светодиоды PD7 и PD4 меняют яркость.

c

```c
                pb = adc_avg * 8 / 1024 + 1;
                if (pb > 8) pb = 8;
```

Преобразуем значение АЦП в pb от 1 до 8. Ограничиваем сверху на случай если АЦП вдруг выдаст ровно 1024.

c

```c
                ind[0]=13;
                ind[1]=11;
                ind[2]=18;
                ind[3]=pb;
```

Обновляем массив ind — индикатор сразу покажет новое значение pb. Таймер T0 сам заберёт это значение при следующем прерывании.

#### Ключевая идея — разделение труда

В программе три уровня:

Прерывания делают срочное — двигают светодиоды, читают АЦП, реагируют на кнопки. Они должны работать быстро и не задерживаться.

Главный цикл делает неспешное — усредняет замеры потенциометра, считает pb, обновляет ind[]. Это не срочно, может подождать несколько миллисекунд.

Переменные mode, adc_ready, adc_value, pb, ind[] — это мост между прерываниями и главным циклом. Прерывание записывает, главный цикл читает. Именно поэтому они объявлены как volatile — это говорит компилятору не оптимизировать эти переменные, потому что они могут измениться в любой момент из прерывания.
