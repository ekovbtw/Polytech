#### Зачем вообще нужен таймер

Процессор выполняет код очень быстро — миллионы операций в секунду. Если просто написать "сдвинь светодиоды" в главном цикле, всё будет мигать с бешеной скоростью. Таймер нужен чтобы делать действия строго через заданные промежутки времени — независимо от того, что делает основной код.

---

#### Режим CTC — считаем до нужного числа

Таймер T0 настроен в режиме CTC (Clear Timer on Compare). Это значит:

- Таймер считает от 0 вверх
- Когда досчитал до OCR0 — вызывает прерывание и сбрасывается обратно в 0
- И так по кругу

Визуально:

```
0 → 1 → 2 → ... → 154 → ПРЕРЫВАНИЕ → 0 → 1 → 2 → ...
                           ↑
                      вызов ISR(TIMER0_COMP_vect)
```

---

#### Как настроен таймер в коде

c

```c
TCCR0 = (1<<WGM01) | (1<<CS01) | (1<<CS00);
OCR0 = 154;
TIMSK |= (1<<OCIE0);
```

Разбираем каждый бит:

WGM01 = 1, WGM00 = 0 — это режим CTC

CS01 = 1 и CS00 = 1 вместе — это делитель 64. Таймер считает не на каждый такт процессора, а на каждый 64-й:

```
8 000 000 / 64 = 125 000 тиков таймера в секунду
```

OCR0 = 154 — таймер считает до 154 включительно, то есть 155 шагов:

```
125 000 / 155 ≈ 800 прерываний в секунду
```

OCIE0 — разрешаем прерывание по совпадению с OCR0

---

#### Что происходит внутри прерывания

Каждые ~1.25 мс вызывается ISR(TIMER0_COMP_vect). Внутри два больших блока:

Если mode == 1 (режим настройки) — обновляем семисегментный индикатор. Про это поговорим на шаге 7.

Если mode == 0 (режим гирлянды) — вот что происходит:

c

```c
garland_tick++;
if (garland_tick >= GARLAND_DIV) {  // каждые 64 прерывания
    garland_tick = 0;
    
    // порт A: сдвиг каждые pa тиков гирлянды
    if (++count_a >= pa) {
        count_a = 0;
        crcl(&cur_a, ha);
    }
    
    // порт B: сдвиг каждые (25/pb) тиков гирлянды
    if (++count_b >= (25 / pb)) {
        count_b = 0;
        crcl(&cur_b, hb);
    }
    
    // порт C: сдвиг каждые pc тиков гирлянды
    if (++count_c >= pc) {
        count_c = 0;
        crcl(&cur_c, hc);
    }
    
    update_garland(); // выводим на порты
}
```

#### Два уровня счёта — важно понять

Здесь есть два вложенных уровня:

```
Уровень 1 — тики таймера (800 в секунду)
    ↓ каждые 64 тика
Уровень 2 — тики гирлянды (12.5 в секунду)
    ↓ каждые pa / (25/pb) / pc тиков гирлянды
Уровень 3 — сдвиг светодиодов
```

Визуально для порта B при pb = 1:

```
Тики таймера:   1..64 → 65..128 → ... → 1536..1600 → сдвиг!
Тики гирлянды:    1        2     → ...      25       → сдвиг!
Время:                                             = 2 секунды
```

#### Функция update_garland

Это просто — берём текущие значения cur_a, cur_b, cur_c и записываем их в порты:

c

```c
void update_garland(void) {
    PORTA = cur_a;
    PORTB = cur_b;
    PORTC = cur_c;
}
```

Что записали в регистр порта — то и горит на светодиодах.
