## Функции 

### 1. crcl() — циклический сдвиг (используется в init и таймере)

Функция делает циклический сдвиг байта на несколько позиций. Циклический - биты которые "вываливаются" с одного конца появляются с другого. 

Принимает **2 аргумента**:
**uint8_t * cur** - указатель на текущее состояние порта(cur_a, cur_b, cur_c), чтобы менять саму переменную.
**int value** - на сколько позиций сдвигать (h).

```
void crcl(uint8_t *cur, int value){
	*cur = ((*cur << value) | (*cur >> (8 - value))) & 0xFF;
}
```
 *cur = ((*cur << value) | (*cur >> (8 - value))) & 0xFF;* - что происходит тут:
 мы сдвигаем влево на столько позиций, сколько требуется, но биты которые "вывалятся" за пределы, должны появится справа, для этого свдигаем вправо на 8-value, и после этих операций над cur, делаем между ними или чтобы все сохранилось, также важно подметить для чего нам 0xFF: uint8_t расширяется до int при операциях, поэтому может стать резко не 8 бит, а больше, для этого используем 0xFF, И с ней дает нам младшие 8 битов, что и будет результатом. Пример:
 
 cur_a = 0b11000000, value = 2
 **Шаг 1 — сдвиг влево на 2:**
```c
0b11000000 << 2 = 0b00000000
```
Оба единичных бита вывалились за пределы 8 бит!

**Шаг 2 — сдвиг вправо на (8-2)=6:**
```c
0b11000000 >> 6 = 0b00000011
```
Вот биты которые вывалились слева появляются справа!

**Шаг 3 — объединяем:**
```c
0b00000000 | 0b00000011 = 0b00000011
```

**Шаг 4 — & 0xFF:**
```c
0b00000011 & 0xFF = 0b00000011
```
Результат:
```
было:  1 1 0 0 0 0 0 0  (PA6, PA7 горят)
стало: 0 0 0 0 0 0 1 1  (PA0, PA1 горят)
```
### 2. update_garland() — вывод cur_a/b/c на порты

Функция просто **выводит текущее состояние трех переменных** cur_a, cur_b, cur_c на физические порты микроконтроллера. Эта функция нужна отдельна, так как мы обновляет cur_a/b/c в разных местах кода, и каждый раз писать три скобки неудобно - лучше вызвать одну функцию. 

```
cur_a → PORTA (светодиоды порта A)
cur_b → PORTB (светодиоды порта B)
cur_c → PORTC (светодиоды порта C)
```
Функция ничего не принимает и ничего не возвращает -> void.

```
void update_garland(void ){
	PORTA = cur_a;
	PORTB = cur_b;
	PORTC = cur_c;
}
```

### 3. init_garlands()  — применяет начальные смещения da/db/dc

Функция применяет начальные смещения da, db, dc к каждому порту при старте программы. Но так как у нас для порта А и В эти значения равны 0, то смещения никакого не будет, а для порта С у нас всего одно смещение, так что можно обойтись без цикла, вызвав функцию один раз. При других параметрах надо было бы делать так:
```
для порта A: повторить da раз → crcl(&cur_a, ha)
для порта B: повторить db раз → crcl(&cur_b, hb)
для порта C: повторить dc раз → crcl(&cur_c, hc)
```
После всех сдвигов вызвать update_garland() чтобы вывести результат на порты. 
```
void init_garlands(){ // — применяет начальное смещение dc
	crcl(&cur_c, hc);
	update_garland();
}
```
### 4. timer_init() — настройка таймеров T0, T1, T2

Функция настраивает **три таймера**. 
#### Timer0 - главный таймер программы. 

Timer0 работает в режиме **СТС - Clear Timer on Compare**, то есть: 
счетчик считает 0, 1, 2 ... 0CR0, когда достигает 0CR0 -> сбрасывается в 0 и вызывает прерывание. Повторяет с точной периодичностью. 

Настраивается через регистр TCCR0: 
TCCR0:    FOC0 | WGM00 | COM01 | COM00 | WGM01 | CS02 | CS01 | CS00 
	    бит7      бит6       бит5       бит4         бит3     бит2    бит1   бит0
	    
Нам нужно WGM01 = 1, WGM00 = 0 -> режим СТС
CS02=0, CS01=1, CS00=1 -> предделитель 64
TCCR0 = (1<<WGM01) | (1<< CS01) | (1<< CS00)

**OCR0** - до какого числа нам считать:
```
OCR0 = 154 → частота прерываний = 8000000 / (64 × 155) ≈ 800 Гц
```
**TIMSK** - разрешение прерывания таймера.
```
TIMSK: [ ... | OCIE0 | ... ]
OCIE0=1 → разрешить прерывание по совпадению Timer0

TIMSK |= (1<<OCIE0)
```

#### Timer1 - ШИМ на PD4 (OC1B)
Сначала разберемся зачем нам ШИМ. Нам нужно, чтобы при прокрутке потенциометра у нас светодиоды на PD4 и PD7 изменяли яркость плавно(без видных ступенек), но микроконтроллер умеет только делать 0В и 5В, то есть гореть и гаснуть, тогда мы используем ШИМ для решения этой проблемы - ШИМ делает быстрое мигания на разных скважностях:
```
100% яркость: ‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾  (всегда HIGH)
75%  яркость: ‾‾‾‾‾‾‾‾‾‾‾‾|____  (75% HIGH, 25% LOW)
50%  яркость: ‾‾‾‾‾‾‾‾|________  (50% HIGH, 50% LOW)
25%  яркость: ‾‾‾‾|____________  (25% HIGH, 75% LOW)
0%   яркость: ________________  (всегда LOW)
```
Мигает очень быстро ~ 31250 раз в сек.

Timer1 - 16 битный таймер, но используем его в 8-битном режиме Fast_PWM=быстрая широтно-импульсная модуляция.

**TCCR1A:** COM1A1|COM1A0|COM1B1|COM1B0|FOC1A|FOC1B|WGM11|WGM10
**COM1A1, COM1A0** — управление ножкой OC1A (PD5)
**COM1B1, COM1B0** — управление ножкой OC1B (PD4)
**WGM11, WGM10** — младшие биты выбора режима: Это только ЧАСТЬ настройки режима! Старшие биты WGM13, WGM12 находятся в TCCR1B.

```
WGM10=1          → Fast-PWM 8-bit (TOP=255)
COM1B1=1,COM1B0=0 → неинвертированный ШИМ на OC1B (PD4)
                    (при совпадении → LOW, при сбросе → HIGH)

TCCR1A = (1<<COM1B1)|(1<<WGM10)
```

**TCCR1B** — что внутри:
```
[ ICNC1 | ICES1 | - | WGM13 | WGM12 | CS12 | CS11 | CS10 ]
   бит7    бит6   бит5  бит4    бит3   бит2   бит1   бит0
```

**ICNC1, ICES1** — захват сигнала
**WGM13, WGM12** — старшие биты выбора режима:
```
Вместе с WGM11, WGM10 из TCCR1A выбирают режим работы
```

**CS12, CS11, CS10** — делитель частоты:
```
CS12=0, CS11=0, CS10=1 → делитель /1
таймер работает на полной частоте 8МГц
```

**Как WGM биты выбирают режим**:
WGM13 | WGM12 | WGM11 | WGM10 | Режим
  0          |   0          |   0          |   0           | Normal (простой счёт)
  0          |   1          |   0          |   0           | CTC
  0          |   1          |   0          |   1           | Fast-PWM 8-бит ← наш!
  0          |   1          |   1          |   0           | Fast-PWM 9-бит
  0          |   1          |   1          |   1           | Fast-PWM 10-бит
  Мы хотим **Fast-PWM 8-бит**:
```
WGM13=0 → не устанавливаем (по умолчанию 0)
WGM12=1 → устанавливаем в TCCR1B
WGM11=0 → не устанавливаем (по умолчанию 0)
WGM10=1 → устанавливаем в TCCR1A
```
Неинвертированный означает: 
```
COM1B = Compare Output Mode for channel B Неинвертированный означает: 
счётчик = 0 → PD4 = HIGH (начало периода) 
счётчик = OCR → PD4 = LOW (совпадение) 
счётчик = 255 → сброс в 0, повтор
```
255  |          /|          /|
	|        /  |        /  |
OCR |----/    |----/    |
	|   /|      |   /|      |
0      |/   |       |__/ |    |
   HIGH LOW HIGH LOW
   ←OCR/255→ = скважность
**OCR1B** : регистр скважности 
```
0   → 0% скважность → PD4 не горит
128 → 50% скважность → средняя яркость
255 → 100% скважность → максимальная яркость

OCR1B = 0  // начальное значение
```
Неинвертированный (COM1B1=1, COM1B0=0)
Инвертированный (COM1B1=1, COM1B0=1)

#### Timer2 - ШИМ на ножке PD7 (OC2):

Timer2 - 8-битный таймер. Настраивается через один регистр:

**TCCR2**:  FOC2|WGM20|COM21|COM20|WGM21|CS22|CS21|CS20 
Нам нужно:
```
WGM21=1, WGM20=1 → Fast-PWM (TOP=255)
COM21=1, COM20=0 → неинвертированный ШИМ на OC2 (PD7)
CS20=1           → делитель /1 (та же частота что T1!)

TCCR2 = (1<<WGM21)|(1<<WGM20)|(1<<COM21)|(1<<CS20)
```
**OCR2**  - регистр скважности, начальное значение 0.

#### Итого что пишем в timer_init()
```
1. Настроить TCCR0  → режим CTC, делитель /64
2. Задать OCR0 = 154
3. Разрешить прерывание TIMSK |= (1<<OCIE0)
4. Настроить TCCR1A → Fast-PWM, ШИМ на OC1B
5. Настроить TCCR1B → Fast-PWM, делитель /1
6. Задать OCR1B = 0
7. Настроить TCCR2  → Fast-PWM, ШИМ на OC2, делитель /1
8. Задать OCR2 = 0
```

```
void timer_init(void){    // — настройка таймеров T0, T1, T2
	TCCR0 = (1<<WGM01) | (1<<CS01) | (1<<CS00); // настраиваем режим СТС
	OCR0 = 154; // до какого числа считать, частота прерываний 8000000 / (64 × 155) ≈ 800 Гц
	TIMSK |= (1<<OCIE0); // разрешить прерывание 
	
	//используем Timer1 в 8-битном режиме Fast-PWM.
	TCCR1A = (1<<COM1B1)|(1<<WGM10);
	TCCR1B = (1<<WGM12)|(1<<CS10);
	
	// регистр сквадности 0% скважности -> PD4 не горит
	OCR1B = 0; // начальное значение
	
	// используем Timer2 (ШИМ на PD7)
	TCCR2 = (1<<WGM21)|(1<<WGM20)|(1<<COM21)|(1<<CS20);
	// регистр скважности 
	OCR2 = 0;  // начальное значение
}
```

### 5. ADC_init() — настройка АЦП

Функция выполняет настройку АЦП.
Настраивает два регистра которые мы уже разбирали:

**ADMUX** — что читаем:
```
REFS0=1  → опорное от AVCC (5В)
MUX2=1, MUX0=1 → канал ADC5 (PA5 — потенциометр)
ADLAR не устанавливаем → правое ориентирование, читаем ADC (10 бит)
```

**ADCSRA** — как работает:
```
ADEN=1  → включить АЦП
ADPS2=1, ADPS1=1 → делитель /64 (как у одногруппника)
ADIE=1  → разрешить прерывание по завершению
ADSC не устанавливаем → запустим вручную когда нужно
```

1. АЦП включается (ADEN=1)
2. Запускается первый замер (ADSC=1) (начинай измерять напряжение на PA5)
3. Через 25 тактов → замер готов
```
Обычный замер (13 тактов):
[выборка 1т][преобразование 10т][запись 2т] = 13

Первый замер (25 тактов):
[инициализация 12т][выборка 1т][преобразование 10т][запись 2т] = 25
```
4. Вызывается ISR(ADC_vect):
   adc_value = ADC;
   adc_ready = 1;
5. Сразу запускается следующий замер (ADATE=1)
6. Через 13 тактов → снова ISR
7. И так бесконечно...

Получаем:
```
void ADC_init(void){       //— настройка АЦП
	ADMUX = (1<<REFS0) | (1<<MUX2) | (1<<MUX0) ;  // опорное от AVCC (5В)

	ADCSRA = (1<<ADEN)|(1<<ADPS2)|(1<<ADPS1)|(1<<ADIE);
}
```
### 6. take_adc()  — запуск одного замера
Просто устанавливает бит ADSC и сбрасывает флаг готовности:

```
ADCSRA |= (1<<ADSC) → запуск замера
adc_ready = 0       → сбрасываем флаг

```
Получаем:
```
void take_adc(void){      // — запуск одного замера
	ADCSRA |= (1<<ADSC); // запуск замера
	adc_ready = 0;       // сбрасываем флаг
}
```

## Зачем нам делать замеры АЦП
По заданию варианта 5а:
```
pb ∈ [1..8] — настраивается потенциометром
```

Потенциометр — это **аналоговое устройство**. Он выдаёт напряжение от 0В до 5В в зависимости от положения ручки.

Микроконтроллер — **цифровое устройство**. Он не понимает напряжение, он понимает только числа.
### Цепочка от ручки до параметра

```
Крутишь ручку потенциометра
        ↓
Напряжение на PA5 меняется (0В..5В)
        ↓
АЦП делает замер → переводит напряжение в число
        ↓
adc_value = 0..1023
        ↓
Пересчитываем в pb:
pb = adc_value * 8 / 1024 + 1
        ↓
pb = 1..8
        ↓
Гирлянда на PORTB меняет скорость
```

### Если бы не было замеров

```
Без АЦП:
Крутишь ручку → ничего не происходит
Микроконтроллер не знает что изменилось напряжение
pb никогда не меняется
```
