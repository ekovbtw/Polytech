### Обработчик 
ISR = Interrupt Service Routine -обработчик прерывания. То есть работает примерно так:
main работает...
    ↓
АЦП закончил замер → прерывание!
    ↓
main ставится на паузу
    ↓
выполняется ISR(ADC_vect)
    ↓
main продолжается с того же места
### 1. ISR(ADC_vect)
```
ISR(ADC_vect){
	adc_value = ADC;
	adc_ready = 1;
}
```
Сохраняем результат из регистра ADC в переменную adc_value. Устанавливаем влаг adc_ready в 1 чтобы main знал что данные готовы. 

После того, как АЦП сделал замер - результат (0-1023) лежит в двух регистрах:
	 ADCL — младшие 8 бит
	 ADCH — старшие 2 бита
Читать их по отдельности неудобно. Поэтому AVR предоставляет специальный объединенный регистр ADC, который автоматически склеивает их в одно 10-битное число:
	ADCH:ADCL = 0b00 10110101  (например число 437)
	ADC       = 437            ← просто читаем одно число

 #### **Зачем adc_value и adc_ready**

Проблема: ISR и main работают независимо. Main не знает когда АЦП закончил замер.

Решение -два флага:
```
adc_value — "что намерили"
adc_ready — "готово ли"
```

Работает как записка:
```
ISR:
  adc_value = ADC;   // записываем результат в "записку"
  adc_ready = 1;     // пишем "готово!"

main:
  if (adc_ready) {   // проверяем "записку"
      // читаем adc_value и используем
      adc_ready = 0; // сбрасываем флаг
  }
```

### Визуально весь процесс

```
Потенциометр крутится
        ↓
Напряжение на PA5 меняется
        ↓
take_adc() запускает замер (ADSC=1)
        ↓
АЦП 64 такта измеряет напряжение
        ↓
Результат готов → вызывается ISR(ADC_vect)
        ↓
adc_value = ADC  (сохраняем число 0..1023)
adc_ready = 1    (сигнализируем main)
        ↓
main видит adc_ready=1
        ↓
пересчитывает в pb/db/bb
        ↓
обновляет индикатор и ШИМ
```

### 2. ISR(TIMER0_COMP_vect)
Таймер0 считывает от 0 до 154 и сбрасывается. Это происходит 800 раз в секунду. Каждый раз вызывается этот обработчик.

**В режиме настройки, когда mode = 1:**
```
if (mode == 1) {
		PORTA = 0x00;                     // 1. гасим ВСЕ индикаторы
		if (++cur_ind > 3) cur_ind = 0;   // 2. переходим к следующему
		PORTC = ~numb[ind[3 - cur_ind]];  // 3. устанавливаем символ
		PORTA = (1 << cur_ind);           // 4. включаем нужный индикатор
	}
```
у нас 4 индикатора, но одновременно может гореть только один. Чтобы казалось что горят все - быстро переключаем их по очереди 800 раз в секунду. Глаз не успевает заметить мигание. 

Для начала гасим все:  PORTA = 0x00;
Если сразу переключить индикаторе  не погасив, старый символ на долю микросек появится и образуется тень, а мы должны ее избегать. 
Гасим → меняем символ → включаем нужный

Далее переходим к следующему, но сначала увеличиваем, если вышли за пределы, сбрасываем в начало. (чтобы образовать круг) 0 → 1 → 2 → 3 → 0 → 1 → ...

Далее устанавливаем символ: 
ind[3 - cur_ind] - индекс символа для текущего индикатора,  номер элемента в списке numb 
cur_ind=0 → ind[3] → правый индикатор (значение pb) 
cur_ind=1 → ind[2] → третий индикатор (пробел) 
cur_ind=2 → ind[1] → второй индикатор (b.) 
cur_ind=3 → ind[0] → левый индикатор (p) 
numb[...] → байт из таблицы символов 
~numb[...]→ инвертируем (общий анод!) 
PORTC → выводим на сегменты

Далее включаем индикатор:
```
PORTA = (1 << cur_ind);
```
```
cur_ind=0 → PORTA = 00000001 → горит индикатор 0
cur_ind=1 → PORTA = 00000010 → горит индикатор 1
cur_ind=2 → PORTA = 00000100 → горит индикатор 2
cur_ind=3 → PORTA = 00001000 → горит индикатор 3
```

**В режиме гирлянды, когда mode - 0**
```
if (mode == 0) { // гирлянда двигаем огонь
		garland_tick++;
		if  (garland_tick >= GARLAND_DIV){
			garland_tick = 0;
			if (++count_a >= pa){
				count_a = 0;
				crcl(&cur_a, ha);
			}
			if (++count_b >= (25/ pb)){
				count_b = 0;
				crcl(&cur_b, hb);
			}
			if (++count_c >= pc) {
				count_c = 0;
				crcl(&cur_c, hc);
			}
			update_garland();
		}
	}
```
Таймер срабатывает слишком часто для гирлянды — 800 раз в секунду это очень быстро. Нам нужно замедлить.

**garland_tick** — считает вызовы до GARLAND_DIV (64):

```
800 вызовов / 64 = 12.5 шагов гирлянды в секунду
```

Когда garland_tick достигает 64 — один **тик гирлянды**:

**count_a** считает тики до pa (=4):

```
Каждые 4 тика → сдвиг порта A
12.5 / 4 = ~3 сдвига в секунду для порта A
```

**count_b** считает тики до pb (настраивается [1..8]):

```
pb=1 → каждый тик → 12.5 сдвигов в секунду (быстро)
pb=8 → каждые 8 тиков → 1.5 сдвига в секунду (медленно)
```

**count_c** считает тики до pc (=6):

```
Каждые 6 тиков → сдвиг порта C
```

После всех проверок → **update_garland()** выводит новые значения на порты.

Вся функция:

```
ISR(TIMER0_COMP_vect) {
	if (mode == 1) {
		PORTA = 0x00;                     // 1. гасим ВСЕ индикаторы
		if (++cur_ind > 3) cur_ind = 0;   // 2. переходим к следующему
		PORTC = ~numb[ind[3 - cur_ind]];  // 3. устанавливаем символ
		PORTA = (1 << cur_ind);           // 4. включаем нужный индикатор
	}
	if (mode == 0) { // гирлянда двигаем огонь
		garland_tick++;
		if  (garland_tick >= GARLAND_DIV){
			garland_tick = 0;
			if (++count_a >= pa){
				count_a = 0;
				crcl(&cur_a, ha);
			}
			if (++count_b >= (25/ pb)){
				count_b = 0;
				crcl(&cur_b, hb);
			}
			if (++count_c >= pc) {
				count_c = 0;
				crcl(&cur_c, hc);
			}
			update_garland();
		}
	}
}
```


### 3. ISR(INT0_vect)
Срабатывает когда нажимается кнопка PD2. Один раз нажал - один раз вызвался. 
Нажал кнопку в режиме гирлянды → переходим в настройку
Нажал кнопку в настройке → возвращаемся в гирлянду

**Если mode - 0 - сейчас гирлянда**: 
```
1. mode = 1          → переключаем режим
2. cur_param = 0     → начинаем с первого параметра pb
3. PORTA = 0         → гасим все светодиоды гирлянды
4. PORTB = 0         → гасим
5. PORTC = 0         → гасим
```
Почему гасим порты? Потому что в режиме настройки PORTA используется для индикатора, PORTC для сегментов — старые значения гирлянды мешают.

**Если mode == 1** (сейчас настройка):
```
1. mode = 0          → возвращаемся в гирлянду
2. cur_b = bb        → применяем новое базовое значение порта B
3. init_garlands()   → применяем все смещения заново
```

Вся функция:
```
ISR(INT0_vect) {
	if (mode == 0) {
		mode = 1;         //→ переключаем режим
		cur_param = 0;    //начинаем с первого параметра pb
		PORTA = 0;    //гасим все светодиоды гирлянды
		PORTB = 0;        // гасим
		PORTC = 0;        // гасим
		
		}
	else {
		mode = 0;
		cur_b = bb;
		init_garlands();
	}
}
```

### 4. ISR(INT1_vect)

Срабатывает когда нажимается кнопка PD3. Работает **только в режиме настройки.**
#### Что должна делать
Циклически переключает между тремя параметрами:
```
cur_param = 0 → pb (частота)
cur_param = 1 → db (смещение)
cur_param = 2 → bb (базовое значение)
cur_param = 3 → снова 0 (pb)
```
Это как нажатие кнопки "следующий параметр".

Важно:
Если mode == 0 (гирлянда) — ничего не делаем, кнопка игнорируется.

```
ISR(INT1_vect){
	if (mode == 1){
		cur_param++;
		if (cur_param > 2) cur_param = 0;
	}
}
```
