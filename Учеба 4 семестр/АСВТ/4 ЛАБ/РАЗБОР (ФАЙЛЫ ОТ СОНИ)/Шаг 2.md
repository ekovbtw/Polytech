### Глобальные переменные

Сначала необходимо объявить параметры для трех портов. Опирайся на таблицу 5а:
```
PORTA: ba=3, ha=2, pa=4, da=0  — все фиксированные
PORTB: bb=0x0F, hb=1, db=0     — фиксированные
       pb=1,             — настраиваемые
PORTC: bc=6, hc=1, pc=6, dc=1 — все фиксированные
```
Для каждого порта нужна переменная **cur_x** - текущее состояние светодиодов на каждом порту. 
Все переменные будут типа uint8_t, так как все переменные положительные и int занимает 16 бит, а uint8_t всего 8 бит, то есть 1 байт памяти. тип переменной volatile говорит нам о том, что переменная может изменятся в прерываниях. 
```
// Порт A — все фиксированные
uint8_t ba = 3;
uint8_t cur_a = 3;
uint8_t ha = 2;
uint8_t pa = 4;
uint8_t da = 0;

// Порт B — частично настраиваемые
uint8_t bb = 0x0F;
uint8_t cur_b = 0x0F;
uint8_t hb = 1;
volatile int pb = 1;
uint8_t db = 0;

// Порт C — все фиксированные
uint8_t bc = 6;
uint8_t cur_c = 6;
uint8_t hc = 1;
uint8_t pc = 6;
uint8_t dc = 1;
```

Расшифровка каждой переменной отдельно для каждого индикатора:
### порт А

**ba = 3;** - параметр b отвечает за то, сколько и какие лампочки горят на порту, в нашем случае ba = 3, то есть на порту А будут гореть 0x00000011 - то есть PA0 и PA1 светодиоды. (1 - горит, 0 - не горит, так как светодиодов 8, проводим аналогию с отдельно каждым битом)

**cur_a = 3;** - привязан к параметру b, отвечает за текущее состояние светодиодов, то есть b - это то какие светодиоды горят изначально, но в бегущем огоньке они еще и двигаются и это надо где-то фиксировать, поэтому заводим эту переменную. 
Например: 0x00000011 -> 0x00001100 -> 0x00110000 - это все фиксируется в cur_a

**ha = 2;** - параметр h отвечает за то, на сколько позиций смещаются горящие светодиоды. (в примере с cur_a показано движение при ha =2)

**pa = 4;** - параметр p отвечает за то, с какой скоростью движутся горящие светодиоды, в этом случае раз в 4 тика. 

**da = 0;** - параметр d отвечает за то, с какого светодиода начнется все действие, в нашем случае с самого первого PA0. 
### порт В

все переменные соответствуют тем же характеристикам, только все происходит на порту В и со своими значениями, разберем их:

bb = 0x0F; - 0x00001111 - PB0, PB1, PB2, PB3 горят

cur_b = 0x0F; аналогично с портом А

hb = 1; - смещение на 1 светодиод

pb = 1; - настраиваемый параметр по нашему варианту от 1 до 8. 

db = 0; - движение начнется с первого светодиода. 
### порт С

bc = 6;  - 0x00000110 PC1 и PC2 горят 
cur_c = 6; - аналогично порту А
hc = 1; - сдвиг на 1 позицию
pc = 6; - раз в 6 тиков происходит сдвиг 
dc = 1; - со второй позиции

### Переменные режима и индикатора

Нужно добавить переменные которые отвечают за **режим работы** и **отображение на индикаторе**. 
1. **mode** - текущий режим программы, 0 -> гирлянда работает, 1 -> режим настройки. Меняется в прерывании INT0.
2. **ind[4]** - массив из 4 элементов, значит каждый = индекс символа в таблице numb для соответствующего индикатора. Меняется в прерываниях.  
3. **cur_ind** — какой индикатор сейчас активен (0,1,2,3). Меняется в прерывании таймера. Для мультиплексирования. ,
```
volatile int mode = 0; // 0 - г работаетб 1 - режим настройки
volatile unsigned int ind[4]= {0, 0, 0, 0}; 
volatile int cur_ind = 0; // 0-3
```

### Переменные гирлянды

**GARLAND_DIV** — делитель. Таймер срабатывает 800 раз в секунду, нам не нужно двигать гирлянду так часто. Каждые 64 тика = один шаг гирлянды.

```
#define GARLAND_DIV 64
```

**garland_tick** — счётчик тиков до следующего шага.
**count_a, count_b, count_c** — счётчики шагов до следующего сдвига каждого порта. Так как у каждого порта своя частота сдвига светодиодов - параметр р, то нужно для каждого порта считать сколько тиков прошло и при нужном значении делать сдвиг. 

```
volatile uint8_t garland_tick = 0;
volatile uint8_t count_a = 0, count_b = 0, count_c = 0;
```
С помощью этого гирлянда будет двигаться не каждый тик а каждые 64 тика, что будет лучше выглядеть и будет уловимо взглядом. 
### Переменные АЦП

```
volatile unsigned int adc_value = 0;
volatile uint8_t adc_ready = 0;
```

**adc_value** — результат последнего замера (меняется в ISR → volatile), число от 0 до 1023:
```
Потенциометр влево  → adc_value ≈ 0
Потенциометр в центре → adc_value ≈ 512
Потенциометр вправо → adc_value ≈ 1023
```

**adc_ready** — флаг "данные готовы" (меняется в ISR → volatile). Если 0 - нельзя еще считывать adc_value, если 1 то можно
