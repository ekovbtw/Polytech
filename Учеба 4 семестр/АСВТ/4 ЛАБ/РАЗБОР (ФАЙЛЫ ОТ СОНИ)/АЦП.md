**АЦП** - 10-битный модуль внутри ATmega32, который превращает напряжение (аналог) в число (цифру). 

## Роль АЦП в этой лабораторной работе 

Ключевая идея в том, что АЦП - это способ управления программой через потенциометр.

Крутим ручку -> меняем напряжение -> АЦП считывает -> программа получает число -> число становится значением параметра гирлянды. 

АЦП используется только в режиме настройки. 

## Как часто работает АЦП

По условию считывание должно идти через прерывание, то есть АЦП работает постоянно или циклически, как только есть новый результат -> вызывается обработчик 

## На практике:

АЦП ATmega32 нужно опорное напряжение для сравнения с входным напряжением (если измеряемое равно опорному, то получаем максимальный код в двоичном виде). Опорное напряжение подаётся на вывод ADRef  или может использоваться внутренний генератор с фиксированным напряжением 2,65 В. 
#### Как он считает:
Внутри используется формула:

ADC = V(in)/V(ref)×1024, где: V(in) — напряжение с потенциометра, V(ref)​ — опорное напряжение, результат — число от **0 до 1023 (10 бит)**

Например, пусть опорное напряжение равно 5В, а на вход подано 2В. Тогда ADC = 2/5 * 1024 = 409.  Это значит, что напряжение 2В будет преобразовано в число 409. 
Если на вход подать 4В, то ADC = 4/5 * 1024 = 819. То есть чем больше входное напряжение, тем больше число на выходе АЦП. 

То есть : **АЦП оценивает не абсолютное значение напряжения, а его отношение к максимальному допустимому уровню**. Получается, что АЦП отвечает на вопрос, какую часть от максимального напряжения составляет входное напряжение. 

#### Вывод AREF, ADRef 

В микроконтроллере есть специальный пин, который называется AREF. Этот вывод нужен для того, чтобы подать опорное напряжение извне. АЦП должен знать, какое напряжение считать "максимумом" (V(ref)), если использовать AREF, то ты сам задаешь этот максимум. 
Пример, если подать на него 3В, то теперь 0В - 0, 3В - 1023, 1.5В - ~512. 

### Регистры работы с АЦП.
### 1. Регистр ADCSRA.
Это регистр управления и статуса АЦП. 

**ADCSRA**:   ADEN | ADSC | ADATE | ADIF | ADIE | ADPS2 | ADPS1 | ADPS0 
        бит7       бит6      бит5   бит4   бит3     бит2      бит1      бит0

**ADEN** - 7 бит, который включает АЦП. 0 - АЦП включен (по умолчанию),  1 - включен и готов к работе. 

**ADSC** - 6 бит, запуск преобразования, записываем 1 - преобразование началось, само становится 0, значит преобразование закончено.

**ADATE** - авто-запуск, 0 - только ручной запуск через ADSC, 1 - АЦП запускается автоматически по событию, источник выбирается в регистре SFIOR.

**ADIF** - флаг завершения, этот бит не трогаем вручную при запуске - сам меняется: 1 - АЦП закончил преобразование, данные ADCL/ADCH готовы, сбрасывается автоматически если используются прерывания. 

**ADIE** - разрешение прерывания, 0 - прерывание по завершению АЦП запрещено, 1 - когда АЦП закончит замер, программа прыгнет на обработчик прерывания.

**АDPS2-0** - делитель частоты. АЦП не может работать на полной частоте процессора, ему нужна частота 50-200 кГц, делитель замедляет тактовую частоту до нужной. Для 10-битной точности нужен делитель 64 или 128. 

### 2. Регистры ADCL и ADCH

Результат АЦП - число от 0 до 1023, для хранения которого нужно 10 бит, но каждый регистр микроконтроллера вмещает только 8 бит (макс. 255). Поэтому результат разрезается на две части и хранится в двух регистрах:

Результат АЦП = 10 бит = 0b10_1101_0110  (например, число 726)
ADCH хранит старшие биты:  0b00000010  (биты 9 и 8)
ADCL хранит младшие биты:  0b11010110  (биты 7..0)
#### Почему ADCL нужно читать первым?

Это аппаратная защита от ошибок. Когда ты читаешь ADCL — микроконтроллер **замораживает** ADCH и не даёт АЦП обновить данные до тех пор, пока ты не прочитаешь ADCH. Если читать в обратном порядке — ADCH и ADCL могут оказаться от **разных преобразований**, и результат будет неверным.

После преобразования, 10-битный результат оказывается в 8-битных регистрах ADCL и ADCH. По умолчанию, младший бит результата находится справа (так называемое правое ориентирование). 

### 3. Регистр ADMUX
Регистр выбора мультиплексора АЦП. 
ADMUX:  REFS1 | REFS0 | ADLAR | MUX4 | MUX3 | MUX2 | MUX1 | MUX0 
         бит7    бит6        бит5    бит4     бит3     бит2      бит1    бит0

Регистр отвечает за 3 вещи: 

1. **REFS1:REFS0 — откуда брать опорное напряжение

Опорное напряжение — это **эталон максимума**. АЦП сравнивает входной сигнал с ним.

```
REFS1 | REFS0 | Источник
  0   |   0   | Внешний пин AREF — ты сам подаёшь нужное напряжение
  0   |   1   | AVCC (питание платы, 5В) ← самый частый вариант
  1   |   0   | Зарезервировано — не использовать
  1   |   1   | Внутренний генератор 2.65В
```

Простой пример — как выбрать линейку:

```
AREF  → ты принёс свою линейку (любой эталон)
AVCC  → используешь стандартную линейку 5В
2.65В → используешь встроенную маленькую линейку
```

Для варианта 5а используем **AVCC**: REFS1=0, REFS0=1

2. **ADLAR — как хранить результат

```
ADLAR = 0 → правое ориентирование (по умолчанию)

  ADCH: [ 0 | 0 | 0 | 0 | 0 | 0 | б9 | б8 ]
  ADCL: [ б7 | б6 | б5 | б4 | б3 | б2 | б1 | б0 ]
  
  Нужно читать оба регистра → получаешь число 0..1023


ADLAR = 1 → левое ориентирование

  ADCH: [ б9 | б8 | б7 | б6 | б5 | б4 | б3 | б2 ]  ← только это читаешь
  ADCL: [ б1 | б0 |  0 |  0 |  0 |  0 |  0 |  0 ]  ← игнорируешь
  
  Читаешь только ADCH → получаешь число 0..255
```

Когда что использовать:

- Нужна **точность** (например, мелкие изменения потенциометра) → ADLAR=0, читаешь оба
- Хватает **грубого значения** (просто левый/правый/середина) → ADLAR=1, читаешь только ADCH

3. **MUX4:MUX0 — с какой ножки читать сигнал

ATmega32 имеет 8 аналоговых входов на PORTA. MUX выбирает один из них:

```
MUX4:MUX0 | Канал | Ножка
  00000   |  ADC0 |  PA0
  00001   |  ADC1 |  PA1
  00010   |  ADC2 |  PA2
  00011   |  ADC3 |  PA3
  00100   |  ADC4 |  PA4
  00101   |  ADC5 |  PA5  ← вариант 5а (потенциометр на PA5)
  00110   |  ADC6 |  PA6
  00111   |  ADC7 |  PA7
```
