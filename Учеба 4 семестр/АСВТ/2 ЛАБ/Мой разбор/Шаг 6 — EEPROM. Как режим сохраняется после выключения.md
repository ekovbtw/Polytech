### Что такое EEPROM и зачем он нужен

Обычная память (SRAM) — где живут все переменные — при выключении питания стирается. Включил снова — всё с нуля.

EEPROM — это отдельная энергонезависимая память. Она сохраняет данные даже без питания. В ATmega32 её 1024 байта. Именно туда мы записываем номер режима, чтобы после перезагрузки гирлянда стартовала с того же режима, на котором была остановлена.

---

### Три функции работы с EEPROM в коде

В коде используется библиотека avr/eeprom.h. Она даёт три готовые функции:

eeprom_read_byte(адрес) — читает один байт из EEPROM по адресу.

eeprom_write_byte(адрес, значение) — пишет байт. Каждая ячейка выдерживает ~100 000 записей.

eeprom_update_byte(адрес, значение) — умная запись: сначала читает что там лежит, и пишет только если значение изменилось. Это экономит ресурс ячейки.

---

### Как это выглядит в коде

```
#define EEPROM_MODE_ADDR  ((uint8_t *)0x00)
```

Это просто адрес — нулевая ячейка EEPROM. Мы сами выбрали куда класть режим. Можно было выбрать любой адрес от 0 до 1023.

Сохранение режима:

```
void eeprom_save_mode(void)
{
    eeprom_update_byte(EEPROM_MODE_ADDR, mode);
}
```

Загрузка режима при старте:

```
uint8_t eeprom_load_mode(void)
{
    uint8_t m = eeprom_read_byte(EEPROM_MODE_ADDR);
    if (m < 1 || m > 3) m = 1;
    return m;
}
```

Проверка if (m < 1 || m > 3) нужна потому что в только что купленном МК все ячейки EEPROM заполнены значением 0xFF = 255. Это явно не режим — поэтому если прочитали мусор, ставим режим 1 по умолчанию.

### Почему запись идёт через main(), а не прямо в ISR?

Запись в EEPROM — медленная операция. Один байт пишется около 3.4 мс. Это огромное время для МК работающего на 8 МГц. Если писать прямо в ISR — всё это время другие прерывания будут заблокированы. Поэтому ISR только ставит флаг need_eeprom_save = 1, а main() уже спокойно делает запись в своём цикле — там никаких ограничений по времени нет.
