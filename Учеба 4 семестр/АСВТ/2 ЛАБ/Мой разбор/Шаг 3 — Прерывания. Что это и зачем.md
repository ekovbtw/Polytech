### Проблема без прерываний

Представь: микроконтроллер крутится в бесконечном цикле и мигает гирляндой. И вдруг пользователь нажал кнопку. Как об этом узнать?

Первый вариант — постоянно проверять кнопку внутри цикла:

```
while(1) {
    мигаю гирляндой...
    проверяю кнопку...
    мигаю гирляндой...
    проверяю кнопку...
}
```

Это называется опрос (polling). Работает, но неудобно: код запутывается, и если где-то стоит задержка — кнопку можно просто пропустить.

### Второй вариант — прерывание

Прерывание — это сигнал, который говорит процессору: "стоп, случилось что-то важное, займись этим прямо сейчас".

Процессор буквально прерывает то, что делал, запоминает своё место, выполняет специальную функцию-обработчик (ISR — Interrupt Service Routine), и потом возвращается обратно — как будто ничего не было.

Вот как это выглядит в виде схемы — процессор спокойно работает, потом прилетает прерывание, он уходит в обработчик и возвращается обратно.

### Как это настраивается в коде

Чтобы прерывания вообще заработали, нужно сделать три вещи.

Первое — выбрать условие срабатывания. В нашем коде:

```
MCUCR = (1<<ISC11)|(1<<ISC10)|(1<<ISC01)|(1<<ISC00);
```

ISC11+ISC10 = 1+1 → INT1 срабатывает по фронту 0→1 (отпускание кнопки). ISC01+ISC00 = 1+1 → INT0 тоже по фронту 0→1. Это значит прерывание срабатывает когда кнопку отпускают, а не нажимают — потому что кнопка подтянута к питанию и при нажатии даёт 0, при отпускании — 1.

Второе — разрешить конкретные прерывания:

```
GICR = (1<<INT1)|(1<<INT0);
```

Третье — сбросить флаги чтобы прерывание не сработало сразу при включении, и разрешить прерывания глобально:

```
GIFR = (1<<INTF1)|(1<<INTF0);
sei();
```

sei() — это одна команда, которая ставит бит в регистре SREG и говорит: "теперь все прерывания разрешены".

### А сам обработчик — это обычная функция?

Почти. Пишется как функция, но с особым синтаксисом:

```
ISR(INT0_vect)
{
    mode++;
    if (mode > 3) mode = 1;
    need_eeprom_save = 1;
    need_update = 1;
}
```

ISR — это макрос из библиотеки avr/interrupt.h. INT0_vect — имя вектора прерывания (из таблицы в методичке). Компилятор сам знает, что эту функцию нужно вызывать когда сработает INT0.
