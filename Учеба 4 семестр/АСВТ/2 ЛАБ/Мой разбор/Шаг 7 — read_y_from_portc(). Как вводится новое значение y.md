### Что происходит когда нажимают PD7

По заданию: нажал PD7 — гирлянда останавливается, можно нажать комбинацию кнопок на PORTC, это станет новым значением y. Отпустил PD7 — гирлянда продолжает с новым y.

Проблема в том что микроконтроллер работает на 8 МГц — он успеет прочитать PORTC тысячи раз пока ты ещё только тянешься к кнопкам. Поэтому нужна специальная последовательность с задержками.

---

### Дребезг кнопки — что это

Когда нажимаешь кнопку, контакты физически "прыгают" несколько миллисекунд перед тем как зафиксироваться. МК видит это как серию быстрых нажатий и отпусканий. Это называется дребезг.

### Разбираем код построчно

```
void read_y_from_portc(void)
{
    _delay_ms(20);
```

Ждём 20 мс после нажатия PD7 — это защита от дребезга самой кнопки PD7.

```
    while ((PINC & 0xFF) == 0) {
        if (!(PIND & (1<<PD7))) return;
    }
```

Крутимся пока ни одна кнопка PORTC не нажата. Если вдруг PD7 отпустили — выходим вообще без изменений. PINC & 0xFF читает все 8 бит PORTC. Если все нули — никто ничего не нажал.

```
    _delay_ms(120);
```

Даём 120 мс чтобы человек успел нажать все нужные кнопки одновременно. Это прямо из методички — именно такая задержка рекомендована для считывания комбинации кнопок.

```
    y = PINC;
```

Считываем всю комбинацию нажатых кнопок как одно 8-битное число. Каждая кнопка — один бит.

```
    while ((PINC & 0xFF) != 0 && (PIND & (1<<PD7)));
```

Ждём пока все кнопки PORTC отпущены или PD7 отпущена. Это защита от повторного считывания — без этого цикл main() мог бы вызвать read_y_from_portc() несколько раз подряд пока кнопка ещё зажата.

```
    _delay_ms(20);
    need_update = 1;
}
```

Ещё 20 мс — дребезг при отпускании. Затем ставим флаг чтобы main() обновил PORTD с новым состоянием.
