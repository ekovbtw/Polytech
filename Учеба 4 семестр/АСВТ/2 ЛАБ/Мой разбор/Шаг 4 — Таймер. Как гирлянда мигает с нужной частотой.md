### Проблема

Гирлянда должна переключаться с частотой 2 Гц — то есть два раза в секунду, каждые 0.5 секунды. Как это сделать?

Первый вариант — delay_ms(500). Но тогда процессор всё это время просто стоит и ждёт — кнопки не читает, ничего не делает. Это плохо.

Второй вариант — таймер с прерыванием. Таймер считает сам по себе, параллельно с основной программой. Когда досчитает до нужного числа — вызывает прерывание. Процессор переключает гирлянду и сразу возвращается обратно.

---

### Как устроен Timer0

Timer0 — это просто счётчик внутри микроконтроллера. Он тикает от тактового сигнала и прибавляет 1 к своему значению. Когда значение совпадает с регистром OCR0 — срабатывает прерывание и счётчик сбрасывается. Это называется режим CTC (Clear Timer on Compare).

### Откуда берётся частота 806 Гц

Тактовая частота МК = 8 МГц = 8 000 000 тиков в секунду. Делитель (prescaler) /64 замедляет это в 64 раза:

```
8 000 000 / 64 = 125 000 тиков в секунду
```

OCR0 = 154, значит счётчик считает от 0 до 154 — это 155 шагов:

```
125 000 / 155 ≈ 806 прерываний в секунду
```

### А при чём тут 2 Гц?

Одно прерывание таймера — слишком часто для переключения гирлянды. Поэтому в ISR ведём свой счётчик тиков:

```
TIMER_TICKS_HALF_PERIOD = 200
```

Каждый раз в прерывании: tick_count++. Когда tick_count дошёл до 200 — переключаем состояние и сбрасываем счётчик.

```
806 прерываний / 200 = ~4 переключения в секунду
```

Но это переключение туда-обратно, значит полный цикл (вкл + выкл) = 2 раза → частота мигания = 2 Гц. Именно то, что нужно по заданию.

### Как настраивается таймер в коде

```
TCCR0 = (1<<WGM01) | (1<<CS01) | (1<<CS00);
OCR0  = 154;
TIMSK |= (1<<OCIE0);
```

TCCR0 — регистр управления таймером. WGM01 = 1 включает режим CTC. CS01 и CS00 вместе = делитель /64. OCR0 = 154 — до какого числа считать. TIMSK — разрешение прерывания по совпадению с OCR0.

### Что происходит в самом ISR таймера

```
ISR(TIMER0_COMP_vect)
{
    tick_count++;
    if (tick_count >= 200) {
        tick_count = 0;
        state ^= 1;          // переключаем 0→1 или 1→0

        uint8_t val = get_value();
        PORTA = val;
        PORTB = val;

        if (state) PORTD |=  (1<<PD4);
        else       PORTD &= ~(1<<PD4);
    }
}
```

state ^= 1 — это XOR с единицей. Если state был 0, станет 1. Если был 1 — станет 0. Простой способ переключать между двумя состояниями.
