Вот этот кусок кода:

```c
MCUCR = (1<<ISC11)|(1<<ISC10)|(1<<ISC01)|(1<<ISC00);
GICR  = (1<<INT1)|(1<<INT0);
GIFR  = (1<<INTF1)|(1<<INTF0);

ISR(INT0_vect)
{
    mode++;
    if (mode > 3) mode = 1;
    need_eeprom_save = 1;
    need_update = 1;
}

ISR(INT1_vect)
{
    if (mode <= 1) mode = 3;
    else           mode--;
    need_eeprom_save = 1;
    need_update = 1;
}
```

#### Зачем нужны внешние прерывания?

Кнопки PD2 и PD3 переключают режим гирлянды. Можно было бы проверять их в основном цикле — но тогда нажатие могло бы пропуститься если программа занята чем-то другим. Внешние прерывания **гарантируют** что нажатие кнопки будет обработано немедленно.

#### Регистр MCUCR — условие срабатывания
```c
MCUCR = (1<<ISC11)|(1<<ISC10)|(1<<ISC01)|(1<<ISC00);
```

Разберём побитово:

|Биты|Прерывание|Значение|Условие|
|---|---|---|---|
|ISC01=1, ISC00=1|INT0|11|нарастающий фронт (0→1)|
|ISC11=1, ISC10=1|INT1|11|нарастающий фронт (0→1)|
Таблица условий:

```
ISCx1 ISCx0   Условие
  0     0     низкий уровень
  0     1     любое изменение
  1     0     спадающий фронт (1→0)
  1     1     нарастающий фронт (0→1)  ← наш вариант
```

Нарастающий фронт — прерывание срабатывает когда кнопка **отпускается** (кнопка к VCC через резистор: нажата=1, отпущена=0... нет, кнопка к VCC: нажата даёт 1). Срабатывает когда напряжение переходит с 0 на 1 — то есть при **нажатии**.

#### Регистр GICR — разрешение прерываний

```c
GICR = (1<<INT1)|(1<<INT0);
```

```
(1<<INT0) = 01000000
(1<<INT1) = 10000000
итого:      11000000
```

Этой строкой мы **разрешаем** прерывания INT0 и INT1. Без неё кнопки PD2/PD3 не будут вызывать прерывания.

#### Регистр GIFR — сброс флагов

```c
GIFR = (1<<INTF1)|(1<<INTF0);
```

Когда происходит прерывание — в GIFR устанавливается флаг. Иногда флаг может быть установлен **до** того как мы разрешили прерывания — например от помех при включении. Эта строка **сбрасывает** эти флаги чтобы прерывания не сработали ложно при старте.

#### ISR INT0 — режим вперёд
```c
ISR(INT0_vect)
{
    mode++;           // увеличиваем режим
    if (mode > 3) mode = 1;  // после 3 → снова 1
    need_eeprom_save = 1;    // сохранить в EEPROM
    need_update = 1;         // обновить PORTD
}
```

Циклически: `1 → 2 → 3 → 1 → 2 → ...`

#### ISR INT1 — режим назад
```c
ISR(INT1_vect)
{
    if (mode <= 1) mode = 3;  // если режим 1 → перейти к 3
    else           mode--;    // иначе уменьшить
    need_eeprom_save = 1;
    need_update = 1;
}
```

Циклически: `1 → 3 → 2 → 1 → 3 → ...`

#### Флаги need_update и need_eeprom_save

Заметь — в ISR мы **не пишем в EEPROM и не обновляем PORTD напрямую**. Вместо этого ставим флаги:
```c
need_eeprom_save = 1;  // попросить main записать EEPROM
need_update = 1;       // попросить main обновить PORTD
```

Почему? Потому что:

- Запись в EEPROM **медленная** — нельзя делать в ISR
- ISR должна быть **максимально короткой**
- Main сам проверит флаги и выполнит нужные действия

В main:
```c
if (need_update) { ... обновить PORTD ... }
if (need_eeprom_save) { ... записать EEPROM ... }
```
