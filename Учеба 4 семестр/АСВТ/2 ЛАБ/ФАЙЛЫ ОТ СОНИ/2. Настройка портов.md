```
DDRA  = 0xFF;
DDRB  = 0xFF;
DDRC  = 0x00;
PORTC = 0x00;
DDRD  = (1<<PD0)|(1<<PD1)|(1<<PD4);
PORTD = 0x00;
```
это кусок из main:
- порт А и В настраиваем как выход, так как на этих портах у нас светится сама гирлянда: К этим портам подключены **светодиоды**. Чтобы управлять светодиодом — нужен выход. Записываем `1` в нужный бит — светодиод горит, `0` — не горит.

- порт С настраивается как вход, так как с этого порта мы будем задавать значение при зажиме кнопки PD7, и у порта С отклчаем pull-up: К PORTC подключены **кнопки** которыми вводим число y. В нашей плате кнопки подключены **не через pull-up** а через внешние резисторы к земле — поэтому: кнопка нажата → бит = **1**, кнопка не нажата → бит = **0**. Поэтому `PORTC = 0x00` — pull-up не нужен.

- PORTD — смешанный (часть вход, часть выход)

Это самый сложный порт — разные ножки делают разные вещи:

|Ножка|Направление|Назначение|
|---|---|---|
|PD0|выход|бит 0 номера режима|
|PD1|выход|бит 1 номера режима|
|PD4|выход|текущее состояние (0/1)|
|PD2|вход|кнопка INT0 (режим вперёд)|
|PD3|вход|кнопка INT1 (режим назад)|
|PD7|вход|кнопка ввода y|
##### DDRD = (1<<PD0)|(1<<PD1)|(1<<PD4);
Разберём побитово:

```
(1<<PD0) = 00000001
(1<<PD1) = 00000010
(1<<PD4) = 00010000
итого:     00010011
```
Биты PD0, PD1, PD4 = **1** (выход), остальные = **0** (вход).

PORTD = 0x00;  // pull-up не включён

PD2, PD3, PD7 — кнопки подключены через внешние резисторы, pull-up не нужен.


### Как номер режима отображается на PD0/PD1

Режим хранится как число 1, 2 или 3. В двоичном:

```
Режим 1 = 01 → PD0=1, PD1=0
Режим 2 = 10 → PD0=0, PD1=1
Режим 3 = 11 → PD0=1, PD1=1
```

В коде это делается так:
```c
pd |= (mode & 0x03);  // берём только два младших бита mode
```
`0x03 = 00000011` — маска, оставляет только биты 0 и 1.
