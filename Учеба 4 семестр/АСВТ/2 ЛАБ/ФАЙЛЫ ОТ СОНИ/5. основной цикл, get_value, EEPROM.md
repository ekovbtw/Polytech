
Вот этот кусок кода:
```c
uint8_t get_value(void)
{
    switch (mode) {
        case 1: return (state == 0) ? 0xFF : 0x00;
        case 2: return (state == 0) ? 0xAA : 0x55;
        case 3:
        default: return (state == 0) ? y : (y ^ 0x80);
    }
}

void update_output(void)
{
    uint8_t val = get_value();
    PORTA = val;
    PORTB = val;
    uint8_t pd = PORTD;
    pd &= ~((1<<PD0)|(1<<PD1)|(1<<PD4));
    pd |= (mode & 0x03);
    if (state) pd |= (1<<PD4);
    PORTD = pd;
}

void eeprom_save_mode(void)
{
    eeprom_update_byte(EEPROM_MODE_ADDR, mode);
}

uint8_t eeprom_load_mode(void)
{
    uint8_t m = eeprom_read_byte(EEPROM_MODE_ADDR);
    if (m < 1 || m > 3) m = 1;
    return m;
}

while (1)
{
    if (need_update) { ... }
    if (need_eeprom_save) { ... }
}
```
#### Функция get_value — что выводить на гирлянду

Эта функция смотрит на текущий **режим** и **состояние** и возвращает байт для портов A и B:
```c
switch (mode) {
    case 1: return (state == 0) ? 0xFF : 0x00;
    case 2: return (state == 0) ? 0xAA : 0x55;
    case 3: return (state == 0) ? y : (y ^ 0x80);
}
```

Оператор `? :` — это сокращённый if/else:

```c
(state == 0) ? 0xFF : 0x00
// если state=0 → вернуть 0xFF
// если state=1 → вернуть 0x00
```

Таблица значений:

| Режим | State 0         | State 1           |
| ----- | --------------- | ----------------- |
| 1     | 0xFF = 11111111 | 0x00 = 00000000   |
| 2     | 0xAA = 10101010 | 0x55 = 01010101   |
| 3     | y = 01010101    | y^0x80 = 11010101 |
#### Функция update_output — обновить все выходы

```c
void update_output(void)
{
    uint8_t val = get_value();
    PORTA = val;   // гирлянда
    PORTB = val;   // гирлянда

    uint8_t pd = PORTD;                        // читаем текущий PORTD
    pd &= ~((1<<PD0)|(1<<PD1)|(1<<PD4));      // очищаем PD0,PD1,PD4
    pd |= (mode & 0x03);                        // записываем номер режима
    if (state) pd |= (1<<PD4);                 // записываем состояние
    PORTD = pd;                                 // выводим
}
```

Почему читаем PORTD перед записью? Потому что в PORTD есть **входные ножки** (PD2,PD3,PD7) — если просто записать новое значение целиком, можно случайно включить pull-up на входах. Поэтому:

```
1. Читаем текущий PORTD
2. Очищаем только нужные биты (PD0,PD1,PD4)
3. Записываем новые значения в эти биты
4. Остальные биты не трогаем
```

Пример с режимом 2, состоянием 1:

```
pd = PORTD         = xxxxxxxx
pd &= ~0b00010011  = xxx0xx00  ← очистили PD0,PD1,PD4
pd |= (2 & 0x03)   = xxx0xx10  ← mode=2=10
pd |= (1<<PD4)     = xxx1xx10  ← state=1
PORTD = pd
```
#### EEPROM — энергонезависимая память
EEPROM сохраняет данные даже после выключения питания. Используем её чтобы запомнить режим.
```c
void eeprom_save_mode(void)
{
    eeprom_update_byte(EEPROM_MODE_ADDR, mode);
}
```

`eeprom_update_byte` — умная функция: сначала читает текущее значение, и **записывает только если оно изменилось**. Это важно потому что EEPROM выдерживает только ~100000 циклов записи.
```c
uint8_t eeprom_load_mode(void)
{
    uint8_t m = eeprom_read_byte(EEPROM_MODE_ADDR);
    if (m < 1 || m > 3) m = 1;  // защита от мусора
    return m;
}
```

При первом включении EEPROM содержит `0xFF` (все биты = 1) — это не валидный режим (1-3), поэтому устанавливаем режим 1 по умолчанию.
#### Основной цикл while(1)
```c
while (1)
{
    if (need_update)
    {
        need_update = 0;
        uint8_t pd = PORTD;
        pd &= ~((1<<PD0)|(1<<PD1)|(1<<PD4));
        pd |= (mode & 0x03);
        if (state) pd |= (1<<PD4);
        PORTD = pd;
    }

    if (need_eeprom_save)
    {
        need_eeprom_save = 0;
        eeprom_save_mode();
    }
}
```

Основной цикл делает только две вещи:

**1.** Если ISR прерывания кнопки установила `need_update` — обновляем PORTD (показываем новый режим на индикаторах).

**2.** Если ISR установила `need_eeprom_save` — сохраняем режим в EEPROM.

Это называется **отложенная обработка** — ISR только ставит флаг, а тяжёлую работу делает main.
