# 1. Какие вообще есть виды памяти в микроконтроллере

У микроконтроллера (например, ATmega32) есть **три разные памяти**:

|память|для чего|
|---|---|
|Flash|хранится программа|
|SRAM|переменные|
|EEPROM|долгосрочное хранение|

Главная из них для программы — **Flash**, а для данных — **SRAM**.

---

# 2. Что такое SRAM (очень просто)

SRAM — это **оперативная память микроконтроллера**.

Это то же самое, что **RAM у компьютера**.

Там хранятся **переменные, которые программа меняет во время работы**.

Пример:

секунды  
минуты  
режим работы  
значение АЦП

Все это **должно где-то храниться** → для этого и используется **SRAM**.

---

# 3. Зачем нужен `.DSEG`

В ассемблере память разделяют на **сегменты**.

`.DSEG` означает:

> **data segment** — сегмент данных.

То есть:

.DSEG

говорит ассемблеру:

> дальше будут **переменные**, которые надо разместить в **SRAM**.

---

# 4. Пример из твоего кода

.DSEG  
time: .BYTE 4

означает:

|часть|что значит|
|---|---|
|`.DSEG`|работаем с SRAM|
|`time`|имя переменной|
|`.BYTE 4`|выделить 4 байта|
