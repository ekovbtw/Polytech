# 4. Что такое стек (очень просто)

Стек — это **специальная область памяти**, где временно хранятся данные.

Он работает по принципу:

LIFO  
Last In – First Out

То есть:

что положили последним  
то достанем первым

---

# 5. Аналогия со стопкой тарелок

Представь стопку тарелок:

   ┌─────┐  
   │  A  │ ← последняя положенная  
   ├─────┤  
   │  B  │  
   ├─────┤  
   │  C  │  
   └─────┘

Если взять тарелку — возьмём **A**.

Это и есть стек.

---

# 6. Где находится стек

Стек хранится в **SRAM**.

SRAM выглядит так:

адрес      память  
-------------------------  
0x0060  
0x0061  
0x0062  
...  
0x085F ← RAMEND

`RAMEND` — это **последний адрес SRAM**.

У ATmega32 это примерно:

RAMEND = 0x085F

---

# 7. Почему стек начинается с конца памяти

Стек растёт **вниз**.

адреса памяти  
  
0x085F  ← начало стека  
0x085E  
0x085D  
0x085C  
...  
0x0060

Причина:

- переменные растут **вверх**
- стек растёт **вниз**

Чтобы они **не столкнулись**.

# 8. Указатель стека (Stack Pointer)

Стек управляется специальным регистром:

SP = Stack Pointer

Он показывает:

где сейчас вершина стека

В AVR он состоит из двух частей:

SPH — старший байт  
SPL — младший байт
