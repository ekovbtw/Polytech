В main, после считывания данных, нужно:

- Создать mutex, event и done_event с правильными параметрами — подумай какой тип сброса нужен для каждого события (автосброс или ручной), и в каком начальном состоянии они должны быть
- Положить первую задачу в очередь — весь массив от 0 до N-1
- Создать все рабочие потоки сразу — функцию потока пока можно оставить пустой заглушкой
- После создания потоков разбудить один из них — в очереди уже есть задач


- done_event — ручной сброс. Когда работа завершена, нужно чтобы все потоки это увидели и никто не пропустил сигнал.
- Событие — автосброс. Потому что одна задача предназначена для одного потока. Разбудил одного — сигнал автоматически сбросился, остальные спят дальше.
