Суть в том, что нужно расставить знаки между элементами массива. Но мы будем сразу считать итоговую сумму. Т.е. С помощью маски.

## Зачем нужна маска
Маска нужна для того, чтобы понять какую комбинацию знаков брать. 
Например: 000: х1 + х2 + х3 + х4 или 010: х1 + х2 - х3 + х4
Т.е. там где 0 - плюс, где 1 - минус 
Важно заметить, что количество комбинаций напрямую зависит от количества чисел в примере. Например, если чисел 4 - то всего можно поставить между ними 3 знака, а значит у нас будет 3 бита (старший, средний, младший), а максимально возможное количество комбинаций из трех битов можно составить $2^3$ , тут нам и поможет маска для решения задачи. Будем брать число и преобразовывать его в двоичное значение, с помощью которого будем расставлять знаки выражения. 

Напишем функцию проверки по маске: 
bool rasstan_zn(unsigned long long mask, int* Array_An, int count_An, int S)
{
    long long result = Array_An[0];
    for (int i = 1; i < count_An; i++)
    {
        if ((mask & (1ULL << (i - 1))) == 0)
        {
            result += Array_An[i];
        }
        else
        {
            result -= Array_An[i];
        }
    }
    if (result == S)
    {
        return true;
    }
    return false;
}
