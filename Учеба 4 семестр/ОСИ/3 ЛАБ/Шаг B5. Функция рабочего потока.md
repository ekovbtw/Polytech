Это главная часть программы. Поток должен в цикле:

- Захватить мутекс и попробовать взять задачу из очереди. [[Про мьютекс]]
- Если задача есть — увеличить active_threads, отпустить мутекс, выполнить задачу
- Если задача выполнена — захватить мутекс, уменьшить active_threads, проверить не пора ли сигналить done_event
- Если задачи нет — проверить не пора ли выходить, если нет — отпустить мутекс и заснуть на WaitForSingleObject(event, …)

Пока не пиши саму сортировку внутри — просто заглушку на её месте. Сосредоточься на логике цикла, мутекса и проверок.

Подумай: когда именно нужно сигналить done_event? Подсказка — это момент когда очередь пуста И никто не работает.

    while (1)
    {

        Task buff = get_task(); // берем задачу
        bool complete = false;
        if (buff.normal == true) // если задача нормальная
        {
            active_threads++; 
            ReleaseMutex(mutex);
            /*потом вызов функции qsort*/
         
            complete = true;
        }
        if (buff.normal == false)
        {
            ReleaseMutex(mutex);
            WaitForSingleObject(event, 50); // каждые 50 мс проверяем есть ли задача 
            complete = false;
        }
        if (complete) // если все прошло
        {
            WaitForSingleObject(mutex, INFINITE);
            active_threads--;
            if (head == tail && active_threads == 0) // проверка на конец
            {
                SetEvent(done_event);
            }
            ReleaseMutex(mutex);
        }

        }
    }
}
