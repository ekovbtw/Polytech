Написать kernel.c, которое:
выводит приветствие;
читает маску букв, сохранённую загрузчиком;
показывает приглашение # ; 
читает команду с клавиатуры;
обрабатывает команды DictOS.

по факту 
Это просто **кусок 32-битного кода**, который:

1. стартует по адресу `0x10000`;
2. умеет писать текст в видеопамять;
3. не использует стандартную библиотеку C;
4. не завершает работу, а крутится в бесконечном цикле.

Это уже основная часть лабораторной: интерактивная ОС с командами. Общие требования к ядру перечислены в вариантах. Команды сборки 

Простой вариант для C: gcc -ffreestanding -m32 -c kernel.c -o kernel.o ld -m elf_i386 -Ttext 0x10000 --oformat binary -e kmain -o kernel.bin kernel.o

Здесь ты пишешь **ядро**, и над тобой ещё нет никакой другой ОС. Поэтому методичка отдельно говорит, что в ядре на C/C++ **недоступны стандартные функции**: ни `printf`, ни `strlen`, ни `memcpy`, ни `memset` и так далее.

ГПТ предложил такое минимальное ядро: 
`extern "C" void kmain()`
`{`
    `volatile char* video = (volatile char*)0xB8000;`

    const char* msg = "HELLO FROM KERNEL";

    for (int i = 0; msg[i] != 0; i++)
    {
        video[i * 2] = msg[i];
        video[i * 2 + 1] = 0x07;
    }

    while (1)
    {
        __asm__ __volatile__("hlt");
    }
`}`

Допустим, под него создадим файл kernel.cpp 
дальше была проблема с его компиляцией, в итоге нужно сделать linker.ld и компилировать таким образом:

linker.ld (без него ядро занимало 4КБ ЧТО ОЧЕНЬ МНОГО для такого ядра)

`ENTRY(kmain)`

`SECTIONS`
`{`
    `. = 0x10000;`

    .text :
    {
        *(.text)
    }

    .rodata :
    {
        *(.rodata)
    }

    /DISCARD/ :
    {
        *(.eh_frame)
        *(.comment)
        *(.note*)
    }
`}`

видимо после компиляции бинарник заполнялся мусорными значениями

сама компиляция выглядит так:
g++ -m32 -ffreestanding -fno-pie -fno-exceptions -fno-rtti -fno-asynchronous-unwind-tables -nostdlib -c kernel.cpp -o kernel.o

ld -m elf_i386 -T linker.ld --oformat binary kernel.o -o kernel.bin

заработает, если сделаете в загрузчике в run_kernel mov cl, 0x01
